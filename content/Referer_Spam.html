title=Referer Spam date=2004-03-26 07:45:47.105 type=post status=published tags=Blog Code ~~~~~~

On rare occassion, I'll see a referer from livexxxcams.com or whatever. I couldn't really do much about it, since the referer list is only stored in the running servlet context, and I couldn't really manipulate it behind the scenes.

I've added a quick little action (DeleteReferer) which allows me to kill any referer I like while I'm logged into my blog.

I've avoided comment spam just by having a unique code base which doesn't function like Movable Type or other common packages.
